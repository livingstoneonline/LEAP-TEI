nigelgbanks 2016-12-8

Took a look into and it seems that the url in the data is invalid in two respects.

  • The apostrophe must be "URL Encoded" to be properly escaped.
  • The URL no longer points to a valid page.

The url for this object has this form in the MODS

<mods:relatedItem type="otherVersion">
  <mods:identifier>Helmsley Parish, http://www.helmsleyparish.org.uk/pics/Helmsley%20David%20Livingstone's%20letter.jpg (accessed 9 September 2015).</mods:identifier>
</mods:relatedItem>

So they apostrophe needs to be "URL Encoded" in the same sense that the spaces are "URL Encoded".

  • Space Character becomes %20
  • Apostrophe becomes %27

The other issue is that the page linked to no longer exists.

http://www.helmsleyparish.org.uk/pics/Helmsley%20David%20Livingstone%27s%20letter.jpg
